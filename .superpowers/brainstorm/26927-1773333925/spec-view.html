🗄️ Data Model Changes

loans table — updated fields

  • pip_portion REAL DEFAULT 2.0 — replaces company_portion
  • investor_portion REAL DEFAULT 5.0 — new
  • nil_portion REAL DEFAULT 7.0 — updated from 7.5
  • interest_rate REAL DEFAULT 14.0 — updated from 15.0

payments table — new columns

  • pip_contribution REAL DEFAULT 0
  • investor_contribution REAL DEFAULT 0
  • nil_contribution — retained

donors table — new

CREATE TABLE donors (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  name            TEXT NOT NULL,
  email           TEXT,
  loan_id         INTEGER REFERENCES loans(id),
  amount          REAL NOT NULL,
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);
-- One donor per loan (1:1). Min investment $1,000,000.
-- GT-only platform, no school_id needed.

athletes table — new columns

  • jersey_number INTEGER
  • unit TEXT CHECK(unit IN ('offense','defense'))
  • email TEXT

Payment calculation

interest_paid         = amount - principal_paid
nil_contribution      = interest_paid * (7 / 14)   // 0.5 — 50%
investor_contribution = interest_paid * (5 / 14)   // ~0.357 — 35.7%
pip_contribution      = interest_paid * (2 / 14)   // ~0.143 — 14.3%

🔌 API Changes

New Endpoints

NEW
GET /api/donors

Returns: id, name, email, amount, pct_of_fund, projected_return (amount×1.05), returns_earned (SUM investor_contribution from their loan's payments)

NEW
GET /api/stats/donors

Returns: total_invested, total_returns_earned, projected_total_return

NEW
GET /api/stats/nil-by-position

Returns: SUM(nil_deals.amount) grouped by athlete position

Updated Endpoints

UPDATED
GET /api/loans

Add donor_display_name (join donors, format "F. Lastname")

UPDATED
GET /api/stats/portfolio

Add pip_revenue, donor_returns; update NIL to 7% split

UPDATED
GET /api/stats/company/:id

Update NIL contribution calc to 7%

UPDATED
POST /api/loans

Accept pip_portion (default 2), investor_portion (default 5), nil_portion (default 7)

Removed Endpoints

GET /api/stats/nil-by-sport
REMOVED
GET /api/stats/nil-by-school
